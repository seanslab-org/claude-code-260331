## Achievements

**1. Vocabulary clarity pass — `docknote-memory-clarity.md` (~12KB)**

Resolved the load-bearing ambiguity in the locked spec: the word *engram* was doing two jobs (a file in `cortex/`, and a sentence vector in `sentence_index`). Same word, different layers, never named explicitly. Three-atom framing introduced — *capture · trace · engram* — to disambiguate. Mental rule: capture is what was said, trace is that it was said and where, engram is what it means.

**2. Five-atom reconciliation — the actual foundation**

Before drafting the final doc, discovered `brain-story.md` already had a different 3-atom framing (*entity · claim · engram*). The clarity-pass framing and the brain-story framing are both partial cuts of the same system. Honest accounting is **five atoms**: capture · trace · entity · claim · engram. Each lives in a distinct layer and serves a distinct role. The final doc uses the full five.

**3. Final design doc — `docknote-memory-design-final.md` (~28KB, 457 lines)**

Consolidated four prior docs (`brain-philosophy.md` 6KB + `brain-story.md` 15KB + `brainmemory-design.md` 68KB + `memory-clarity.md` 12KB = 101KB) into one load-bearing 28KB spec. 14 sections: principles · five atoms · storage · Brain API · write path · read path · confirmation · UX · privacy · phase scope · acceptance · open questions · references · what's next.

Source docs retained as design history. The new file is the load-bearing one going forward.

**4. Concretization for Sean — "what does this ship as"**

Answered the disk-shape question: markdown directory at `~/Docknote/brain/` (truth, git-versioned), regenerable SQLite at `~/Library/Application Support/Docknote/brain.db` (cache), Rust module inside the existing Tauri binary (no separate process, no daemon, no MCP). Year-of-use sizing: ~80MB without audio, ~10GB with.

**5. Replacement audit**

Memory v8 is mostly additive. Three under-the-hood replacements:
- flat `~/Docknote/meetings/<id>/` → `brain/raw/` + `cortex/` (one-time migration on first launch)
- `catalog.jsonl` → `cortex/index.md` (Karpathy navigation file)
- Ask AI's 2-phase Selection→Answer pipeline → 5-stage read path (same Tauri command, new engine)

No user-facing surface fully replaced — Memory is a sibling lens to Notes, not a takeover. HiDock, pipeline queue, Notes/Whispers/Todos/Settings UIs, vLLM backend, OpenAI fallback, Keychain — all unchanged.

## Discoveries

**Lock-then-consolidate is a real pass.** Prior session marked the 68K spec "design-locked" with 15 numbered decisions. But Sean reported "memory itself is still not 100% sure." Diagnosis: the design *was* locked, but it had accumulated 101KB of layered drafts with vocabulary collisions and decision-log reasoning columns that were design-process exhaust. Locking the architecture doesn't make it clear; that's a separate consolidation pass.

**The "engram" word collision was the load-bearing unclarity.** Once named (cortex page vs sentence-vector index entry), the read path, the galaxy UI, and the four people-lens queries all stopped feeling slippery. No architectural change required — the 15 decisions in §15 still hold. Pure vocabulary unification fixed it.

**Three new open questions surfaced** that the spec's §14 didn't capture:
- Galaxy plotting unit (sentence-level smear vs meeting-centroid dot — recommended meeting-level)
- Meeting-engram body regeneration boundary (frontmatter authoritative, body regenerable from `raw/`)
- Person/topic-engram body regeneration boundary (both authoritative — the user's confirmed wording is itself the encoding; principle 2 binds)

The third is the load-bearing one. It's where principle 2 starts to *constrain* what's regenerable.

**The five-atom inventory was the union, not a redesign.** Both prior framings (capture/trace/engram from clarity, entity/claim/engram from story) were correct on their own axis. Neither was complete. Going to five wasn't adding complexity — it was naming what was already there.

---

## Decisions taken

- **Final doc shape**: 14 sections, no decision-log reasoning columns, no "earlier draft said X" trace, no tier-by-tier prior-art tables, no sibling-project deep dives. References get one line each.
- **Five-atom inventory** is canonical going forward (capture · trace · entity · claim · engram).
- **Galaxy plots one dot per meeting**, positioned at the centroid of its traces. Sentence-level retrieval is one click deep on the per-meeting view. (Tentative — Sean to confirm.)
- **Meeting-engram body is regenerable from `raw/`**; frontmatter is authoritative.
- **Person/topic-engram body and frontmatter are both authoritative** (not regenerable from traces alone).
